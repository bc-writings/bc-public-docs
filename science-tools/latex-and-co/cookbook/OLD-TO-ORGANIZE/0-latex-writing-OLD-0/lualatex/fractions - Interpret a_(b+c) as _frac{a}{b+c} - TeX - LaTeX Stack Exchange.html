    If you're in a position to use LuaLaTeX, you may achieve your formatting objective quite easily by setting up a Lua function that operates as a preprocessor on the input stream. In the following solution, a Lua function called autofrac is set to operate on three types of expressions:

    • <single letter or numeral>/<single letter or numeral>
    • <single letter or numeral>/(<some expression enclosed in matching round parentheses>)
    • (<some expression enclosed in matching round parentheses>)(<some expression enclosed in matching round parentheses>)

    Observe that whitespace is not allowed on either side of the / character. (This could be relaxed, if desired.)

    The Lua function is activated by running \autofracOn, and it is deactivated by running \autofracOff.

    Two caveats are in order:

    • It is assumed that expressions that match one of the three criteria stated above will occur in math mode only. If you're in the habit of abbreviating "Air Conditioning" as "A/C", say, don't be surprised that LaTeX will issue a warning about \frac{A}{C} occurring while not in math mode.

    • If your document contains expressions such as \theta/3 or \sin(\theta)/(a+b), be sure to include whitespace before or after the / (division) symbol in order to prevent the expressions from being transformed to \thet\frac{a}{3} (probably creating an error message) and \sin\frac{\theta}{a+b} (probably no error or warning message, but clearly not correct), respectively. With \theta/ 3 or \sin(\theta) /(a+b), you'll be safe. Conversely, if you do intend to produce \frac-type notation, you should change the inputs to (\theta)/(3) and (\sin(\theta))/(a+b) so that the expressions fully satisfy the third expression type given above) or -- better yet, in my opinion! -- just use \frac from the outset.

    \documentclass{article}
\usepackage{luacode} % for 'luacode' environment
\begin{luacode}
    -- The main function
function autofrac ( s )
   s = s:gsub ( "(\%w)/(\%w)" ,     makefracA ) -- 'a/b' case
   s = s:gsub ( "(\%w)/(\%b())" ,   makefracB ) -- 'a/(...)' case
   s = s:gsub ( "(\%b())/(\%b())" , makefracC ) -- '(...)/(...)' case
   return s
end
-- Three auxiliary functions
function makefracA ( u , v ) 
   return "\\frac{"..u.."}{"..v.."}"
end
function makefracB ( u , v ) 
   return makefracA ( u , v:sub(2,-2) ) -- strip off parentheses
end
function makefracC ( u , v ) 
   return makefracA ( u:sub(2,-2) , v:sub(2,-2) ) -- strip off parentheses
end
    \end{luacode}
%% LaTeX macros to activate and deactivate 'autofrac':
\newcommand\autofracOn{\directlua{luatexbase.add_to_callback(
   "process_input_buffer", autofrac, "autofrac" )}}
\newcommand\autofracOff{\directlua{luatexbase.remove_from_callback(
   "process_input_buffer", "autofrac" )}}

\begin{document}
\autofracOn % Activate the Lua function
$\displaystyle 1/3 \quad a/b \quad x/(y+z) \quad (x+1)/(x(x+2))$

\bigskip
\autofracOff % Deactivate the Lua function
$\displaystyle 1/3 \quad a/b \quad x/(y+z) \quad (x+1)/(x(x+2))$
\end{document}

    LaTeX is one of formats of TeX. And TeX has \over primitive. The usage is $numerator\over denominator$ or $...{numerator\over denominator}...$. The LaTeX macro \frac is defined (roughly speaking) by \over primitive: \def\frac#1#2{{#1\over#2}}.

    LaTeX does not hide TeX primitives, so you can write:

    Writing out $x+1\over(x)(x+2)$ seems much easier than writing $\frac{x+1}{(x)(x+2)}$.

    You can set the / character as "math active character" and write:

    {\catcode`/=13 \global\let /=\over}
\mathcode`/="8000

Writing out $x+1/(x)(x+2)$ seems much easier than writing $\frac{x+1}{(x)(x+2)}$.

    Of course, if the scope of the fraction is less than whole formula, you must use {} in order to give this scope, for example ${a/b}=c$, as mentioned in the Don Hosek's comment.

    As you can see from Mico's answer, it is really hard to crack down on all the corner cases that come with an approach using textual replacements. In general parsing such constructs requires a finite state machine to correctly render, e.g.

    f(x)/g(x)

    The syntax you are describing reminds me a lot of AsciiMath, a language to write mathematical formulae with only very few special symbols. I don't know of an implementation of AsciiMath in LaTeX, but there exists one in ConTeXt. This might be closer to what you are looking for. However, I'd like to quote Hans Hagen, the implementer of this code, from his talk at TUG 2015 (from memory)

    “Writing AsciiMath is fun. You never know what output you will get.”

    \usemodule[asciimath]

\unexpanded\def\stopasciimath{\stopasciimath}
\unexpanded\def\startasciimath#1\stopasciimath{%
  \startformula
     \asciimath{#1}%
  \stopformula
}

\starttext

\startasciimath
    a/b * alpha/omega * f(x)/g(x) * (x+1)/(a*(b+c))
\stopasciimath

\stoptext

    You can actually use ConTeXt's AsciiMath in LaTeX with surprisingly little effort (but probably with limitations).

    \documentclass{article}
\usepackage{amsmath}
\usepackage{unicode-math} % you have to use this for AsciiMath
\usepackage{environ}
\usepackage{luacode}
\begin{luacode*}
    xml = xml or {}
lxml = lxml or {}
moduledata = moduledata or {}
statistics = statistics or { register = function() end }
require("x-asciimath")

function asciimath(str)
    local texmath = moduledata.asciimath.convert(str)
    assert(texmath) -- sledgehammer error handling
    tex.sprint(texmath)
end
    \end{luacode*}

% Some aliases from ConTeXt
\let\lparent(
\let\rparent)

\protected\def\doasciimath#1{%
    %\enableautofences % No straightforward LaTeX equivalent
    \mathdelimitersmode="16 % Some magic number from ConTeXt
    \directlua{asciimath("\luaescapestring{\detokenize\expandafter{\expanded{#1}}}")}%
}

\NewEnviron{asciimath}{%
    \begin{math}%
        \doasciimath{\BODY}%
    \end{math}%
}

\NewEnviron{asciidisplaymath}{%
    \begin{displaymath}%
        \doasciimath{\BODY}%
    \end{displaymath}%
}

\begin{document}

\begin{asciimath}
    a/b * alpha/omega * f(x)/g(x) * (x+1)/(a*(b+c))
\end{asciimath}

\begin{asciidisplaymath}
    a/b * alpha/omega * f(x)/g(x) * (x+1)/(a*(b+c))
\end{asciidisplaymath}

\end{document}
