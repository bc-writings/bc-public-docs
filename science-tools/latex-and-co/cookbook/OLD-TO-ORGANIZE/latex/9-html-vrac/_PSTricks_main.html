Important: PSTricks packages are divided into the core and contributed ones. The core is pstricks.

Frequently done errors ...

  1. Using PSTricks together with TikZ causes a problem after the last update of the dvi driver of TikZ when using package auto-pst-pdf. Here is an example of how to use the package without a problem.
  2. Do not choose pstricks.tex as a file name, it is the name of the main PSTricks package! Windows users pay attention here, there is no difference in lower or upper case letters.
  3. Missing number, treated as zero This is the default error message when you are using symbolic node names without enabled \SpecialCoor after loading the pst-node package!
  4. PSTricks is PostScript Tricks, viewing your PS or PDF output with a PS or PDF viewer is appropriate. Nearly all DVI-viewers cannot show rotations or movings, hence not the correct output.
  5. You cannot run your files with pdftex/pdflatex, use lualatex instead or the sequence latex->dvips->ps2pdf, or see topic pdfoutput
  6. If you get a rotated page then use always ps2pdf -dAutoRotagePages=/None .... See also here for Windows user
  7. linestyle=dashed doesn't work after an update of pstricks.tex. You should update also pstricks.pro and pstricks.sty.
  8. Error message: no room for a new dimen. See UK TeX-FAQ for what to do ...
  9. \pspicture[0.5] ... doesn't work anymore! Use instead [shift=*] Read the docs!
  10. If you have a problem with the optional argument [pdf], then use always
    \usepackage{auto-pst-pdf,pstricks} or \usepackage{auto-pst-pdf,pstricks-add} or ...
